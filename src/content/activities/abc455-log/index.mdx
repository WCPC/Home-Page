## 今週の出題

### ABC455-A

<details>
	<summary>問題文を表示</summary>
	<div style="background-color: #f5f5f5; padding: 15px; border-radius: 5px; border: 1px solid #ddd;">

		**問題文** <br/>
		整数 A, B, C が与えられます。A &ne; B かつ B = C であるならば `Yes` を、そうでないならば `No`
		を出力してください。<br/>

		**制約** <br/>
		- 1 &le; A, B, C &le; 9 <br/>
		- 入力される値は全て整数 <br/>

		**入力** <br/>
		A B C

		**出力** <br/>
		問題文中の指示に従って `Yes`, `No` のいずれかを出力せよ。

	</div>
</details>

A問題はこれくらいであってくれって感じの簡単な判定問題でしたね．
bool を そのまま Yes/No で出力するメソッドを用意しているので，`out.println(A != B && B == C);` で瞬殺．
私は**23秒AC**で言語別1位に！！

### ABC455-B

<details>
	<summary>問題文を表示</summary>
	<div style="background-color: #f5f5f5; padding: 15px; border-radius: 5px; border: 1px solid #ddd;">

		**問題文** <br/>
		H 行 W 列のグリッドがあります。各マスは白 (`.`) または黒 (`#`) で塗られています。<br/>
		グリッドの長方形領域のうち、点対称に塗られているものの個数を求めてください。<br/>

		**制約** <br/>
		- 1 &le; H, W &le; 10 <br/>
		- H, W は整数 <br/>
		- S_i は `.`, `#` からなる長さ W の文字列 <br/>

		**入力** <br/>
		H W <br/>
		S_1 <br/>
		S_2 <br/>
		⋮ <br/>
		S_H <br/>

		**出力** <br/>
		答えを出力せよ。

	</div>
</details>

B問題では珍しくめんどくさい問題でした．．
左上の座標を決定するループ，長方形のサイズを決定するループ，判定のループで計6重ループで解けます．
Javaならループにラベルを付与できるので，bool変数を使わないで書くこともできます．
解説には `O(HW²)` で解く方法もあるそうな．．．

### ABC455-C

<details>
	<summary>問題文を表示</summary>
	<div style="background-color: #f5f5f5; padding: 15px; border-radius: 5px; border: 1px solid #ddd;">

		**問題文** <br/>
		整数列 A = (A_1, A_2, ..., A_N) が与えられます。<br/>
		次の操作をちょうど K 回行った後の A の各要素の和として考えられる最小値を求めてください。<br/>
		- 整数 x を選び、A_i = x なる全ての i について A_i を 0 に置き換える。<br/>

		**制約** <br/>
		- 1 &le; K &le; N &le; 3 &times; 10^5 <br/>
		- 1 &le; A_i &le; 10^9 <br/>
		- 入力される値は全て整数 <br/>

		**入力** <br/>
		N K <br/>
		A_1 A_2 ... A_N <br/>

		**出力** <br/>
		答えを出力せよ。

	</div>
</details>

Mapの練習問題です．
`HashMap` で各要素の出現回数を記録していき，`key * value` の配列を作成，ソートして `配列のサイズ-K` 個の総和をとるだけで解けます．
マルチセット（要素に対してその個数を value に持つ Map）を読み込む関数などがあれば簡潔に書けます．

### ABC455-D

<details>
	<summary>問題文を表示</summary>
	<div style="background-color: #f5f5f5; padding: 15px; border-radius: 5px; border: 1px solid #ddd;">

		**問題文** <br/>
		カードが N 枚、カードの山が N 個あります。初め、山 i にはカード i のみがあります。<br/>
		各操作で「カード C_i とその上に積まれているカード」を順序を保ってカード P_i の上に移動します。<br/>
		全ての操作後、各山に積まれているカード枚数を求めてください。<br/>

		**制約** <br/>
		- 1 &le; N, Q &le; 3 &times; 10^5 <br/>
		- 1 &le; C_i, P_i &le; N <br/>
		- 各操作直前で C_i と P_i は異なる山にあることが保証される <br/>
		- 各操作直前で P_i はある山の一番上にあることが保証される <br/>
		- 入力される値はすべて整数 <br/>

		**入力** <br/>
		N Q <br/>
		C_1 P_1 <br/>
		C_2 P_2 <br/>
		⋮ <br/>
		C_Q P_Q <br/>

		**出力** <br/>
		最終的に山 i に積まれているカード枚数を A_i として、A_1, A_2, ..., A_N を空白区切りで出力せよ。

	</div>
</details>

今回もこのD問題が速く解けるかが大事でした．連結リストの構造を分かっていればすぐ思いつきそうな問題です．
この問題は，あるカードが現在どこのタワーにいるかを覚える必要がありません．覚える必要があるのは，ある要素につき前後の要素だけで十分です．
私はここまでしか解けませんでしたが，何とか速解きでレートは上がりました．Eを解かないことには水色に上がれそうにありません．．

### ABC455-E

<details>
	<summary>問題文を表示</summary>
	<div style="background-color: #f5f5f5; padding: 15px; border-radius: 5px; border: 1px solid #ddd;">

		**問題文** <br/>
		`A`, `B`, `C` の 3 種類の文字のみからなる長さ N の文字列 S が与えられます。<br/>
		非空部分文字列のうち、`A`, `B`, `C` の出現回数が相異なるものの個数を求めてください。<br/>
		同じ文字列でも取り出す位置が異なれば別の部分文字列として数えます。<br/>

		**制約** <br/>
		- 1 &le; N &le; 2 &times; 10^5 <br/>
		- |S| = N <br/>
		- N は整数 <br/>
		- S は `A`, `B`, `C` のみからなる <br/>

		**入力** <br/>
		N <br/>
		S <br/>

		**出力** <br/>
		答えを 1 行で出力せよ。

	</div>
</details>

残された80分を使ったにも関わらず，更新ロジックにバグがあり解けませんでした．．解法自体は簡単で，命題の対偶を求めることで解けます．
i番目までの A, B, C の出現回数の差に着目し，出現回数の差が同じパターンのものを数え上げるだけです．
反省として，JavaではPair型を扱おうと思うと非常に冗長なコードを記述する必要があり，実装を複雑化させてしまいました．
また，HashMapも少し冗長なコードになりやすく苦労しました．そのためコンテスト後に複数キーに対応した自前のHashMapをライブラリ化することでAC出来ました．
